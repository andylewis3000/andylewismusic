# SEO reference

How search-engine optimization is implemented, and what to check when adding
content.

## Automatic on every page

- **`<title>` + meta description** — from the page's `title`/`description`
  props, falling back to site defaults. (`src/components/seo/SEO.astro`)
- **Canonical URL** — absolute, derived from the route.
- **Open Graph + Twitter cards** — title, description, URL, type, locale, and a
  resolved absolute image (`summary_large_image` when an image exists).
- **JSON-LD structured data** — a single `@graph` per page containing:
  - `WebSite` and `Person` (Andy as musician, with `sameAs` social profiles) on
    **every** page.
  - `MusicAlbum` (+ tracks) and `BreadcrumbList` on **release** pages.
  - `MusicEvent` for every upcoming show on the **events** page.
  - `BlogPosting` (+ `BreadcrumbList`) on **article** pages.
- **RSS auto-discovery** `<link>` and **sitemap** `<link>`.

## Site-wide files

- **`/sitemap-index.xml`** — generated by `@astrojs/sitemap`; the dev
  `/styleguide` page is filtered out.
- **`/robots.txt`** — allows everything except `/admin/`, points to the sitemap.
- **`/rss.xml`** — the News feed (`@astrojs/rss`).

## Content checklist (per item)

When adding content in the CMS, for best SEO:

1. **Description** — write a natural 120–160 character summary. It's the meta
   description and the social-share text.
2. **Image + alt text** — always add alt text; it feeds both accessibility and
   image SEO. The image also becomes the social-share preview.
3. **Titles** — clear and specific; they become the `<title>` and `og:title`.
4. **Dates** — accurate publish/event dates power `datePublished` and event
   rich results.
5. **Per-item SEO overrides** — the optional `seo` fields (title/description/
   image/noindex) let you override the defaults for any entry.

## Verifying

- **Rich results:** paste a URL into Google's Rich Results Test after launch.
- **Structured data shape:** every page embeds one `application/ld+json` block;
  validate at [validator.schema.org](https://validator.schema.org/).
- **Social previews:** check with the Facebook Sharing Debugger / X Card
  Validator once the domain is live (OG images need a real deployed URL).

## Notes

- OG/Twitter images resolve to **absolute** optimized URLs — they only render a
  real preview once the site is deployed to `andylewismusic.com` (and once real
  images exist; until then the default OG image path is used if present).
- The production origin is set once in `astro.config.mjs` (`site`). Everything
  SEO-related derives from it.
